- Show the application title "CRM Dashboard"
- Highlight the navigation tabs at the top

### SCENE 4: EMAIL SEARCH WITH SECURITY (1:30 - 2:15)
**Duration:** 45 seconds  
**Screen:** Email Search tab

**NARRATION:**
"The Email Search feature requires Outlook authentication for security. Notice the blocked state - this is one of our security guard rails. Let me click 'Connect Outlook' to demonstrate. There's also a 30-minute session timeout for added security. Let me click 'Search All' to find email threads for all change requests. The system automatically generates observations based on email patterns, which you can add directly to your comments."

**ON-SCREEN ACTIONS:**
- Show blocked state with large icon
- Click "Connect Outlook" button (demo mode)
- Show connected state with user email
- Click "Search All" button
- Show search results with email threads
- Highlight an observation and click "Add to Comments"

**KEY VISUAL:** Security features in action - authentication required, rate limiting notice

### SCENE 6: EMAIL DRAFT WITH SECURITY (2:45 - 3:30)

**Screen:** Email Draft tab

## 🎯 RECORDING TIPS

### Before Recording:
1. **Clear browser cache** and restart the application
2. **Close unnecessary tabs** and applications
3. **Set browser zoom to 100%**
4. **Disable browser extensions** that might interfere
5. **Test audio levels** - speak clearly and at consistent volume
6. **Prepare sample data** - ensure mock data is loaded

### During Recording:
1. **Speak slowly and clearly** - pause between sections
2. **Use smooth mouse movements** - avoid jerky cursor motion
3. **Highlight important elements** - use cursor to point
4. **Pause after clicks** - allow UI to respond
5. **Maintain consistent pacing** - don't rush

### Recording Settings:
- **Resolution:** 1920x1080 (Full HD)
- **Frame Rate:** 30 FPS minimum
- **Audio:** 44.1 kHz, mono or stereo
- **Format:** MP4 (H.264 codec)
